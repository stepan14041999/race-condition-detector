Reports when this reference escapes during object construction, or when overridable methods are called from constructors.

Why this is a problem:

  • When this escapes during construction, other threads may observe a partially initialized object
  • Calling overridable methods in constructors allows subclass methods to access uninitialized fields
  • This violates the "safe publication" principle from the Java Memory Model

Example (Before - this escape):


public class EventSource {
    public EventSource(EventListener listener) {
        listener.onCreated(this);  // ⚠ 'this' escapes before construction completes
    }
}

Example (Before - overridable method call):


public class Base {
    protected String name;

    public Base() {
        initialize();  // ⚠ Overridable method called in constructor
    }

    protected void initialize() {
        // Subclass override may access uninitialized fields
    }
}

Example (After - using factory method):


public class EventSource {
    private EventSource() {
        // Private constructor
    }

    public static EventSource create(EventListener listener) {
        EventSource source = new EventSource();
        listener.onCreated(source);
        return source;
    }
}
