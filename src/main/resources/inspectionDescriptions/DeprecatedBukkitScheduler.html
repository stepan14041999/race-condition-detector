Reports usage of deprecated BukkitScheduler API in Folia projects.

Why this is a problem:

  • Folia does not support the legacy BukkitScheduler API
  • Methods like runTask(), runTaskLater(), runTaskTimer() will throw exceptions
  • Folia requires using the new region-aware scheduler APIs

Example (Before):


public class MyPlugin extends JavaPlugin {
    @Override
    public void onEnable() {
        Bukkit.getScheduler().runTaskTimer(this, () -> {  // âš  Deprecated in Folia
            // Do something
        }, 0L, 20L);
    }
}

Example (After - using GlobalRegionScheduler):


public class MyPlugin extends JavaPlugin {
    @Override
    public void onEnable() {
        Bukkit.getGlobalRegionScheduler().runAtFixedRate(this, task -> {
            // Do something
        }, 1L, 20L);
    }
}

Example (After - using RegionScheduler for location-specific tasks):


public void scheduleAt(Location location) {
    Bukkit.getRegionScheduler().runDelayed(plugin, location, task -> {
        // Runs on the region thread for this location
    }, 20L);
}

Example (After - using AsyncScheduler for async tasks):


public void doAsyncWork() {
    Bukkit.getAsyncScheduler().runNow(plugin, task -> {
        // Heavy computation or I/O
    });
}
